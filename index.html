File Permissions in Linux

Project Description

For this project I interacted with Linux file permissions to manage the authorizations within the /home/researcher2/projects directory. I practiced checking file and directory details, interpreting Linux permission strings, and modifying permissions with the chmod command.

Check File and Directory Details

Command used:

ls -la projects

This shows hidden files, directory contents, permissions, owner, group, size, and timestamps.

Permissions

File/DirPermissionsOwnerGroup
project_k.txt-rw-rw-rw-researcher2research_team
project_m.txt-rw-r-----researcher2research_team
project_r.txt-rw-rw-r--researcher2research_team
project_t.txt-rw-rw-r--researcher2research_team
.project_x.txt-rw-w-----researcher2research_team
drafts (dir)drwx--x---researcher2research_team

Example Breakdown (project_t.txt)

- First character (-) = regular file
- Next three (rw-) = user can read and write
- Next three (rw-) = group can read and write
- Last three (---) = others have no permissions

Change File Permissions

For project_k.txt, remove write for others:

chmod o-w /home/researcher2/projects/project_k.txt

Hidden File Permissions

For .project_x.txt, read-only for user and group:

chmod 440 /home/researcher2/projects/.project_x.txt

or

chmod u-w /home/researcher2/projects/.project_x.txt

Directory Permissions

For drafts directory, full access only for user:

chmod 700 /home/researcher2/projects/drafts

or

chmod u=rwx,g=,o= /home/researcher2/projects/drafts

Summary

In this project, I reviewed file and directory permissions, updated them to meet security requirements, secured hidden files, and restricted directories to the owner.
